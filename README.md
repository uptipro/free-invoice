# Upti Invoice

A minimal, professional invoicing web application built with React and Vite.  
Features include creating invoices with line items, automatic totals, signature capture, and export to PDF/Excel.

This project now includes an Express + PostgreSQL backend that stores invoice records after the user confirms privacy policy and downloads a PDF.

## Getting started

```bash
npm install
cp .env.example .env
docker compose up -d
npm run dev
```

Open http://localhost:5173 in your browser.

The API server runs on http://localhost:4000.

## Use Supabase Postgres Only (no local DB)

You can connect this app directly to Supabase Postgres and skip Docker.

1. Create a Supabase project.
2. Go to Project Settings > Database.
3. Copy the Connection string (URI). Use the Transaction or Session pooler URI for external apps.
4. Set your .env values like this:

```bash
VITE_API_BASE_URL=http://localhost:4000
DATABASE_URL=<SUPABASE_DATABASE_CONNECTION_URI>
DB_SSL=true
DB_SSL_REJECT_UNAUTHORIZED=false
```

5. Start the app:

```bash
npm run dev
```

Do not run `docker compose up -d` when using Supabase DB only.

## Supabase SDK Mode (recommended for this project)

This codebase is Vite + Express (not Next.js), so Next middleware/server-component files are not required.

If direct Postgres URI connectivity fails, backend can store invoice records through Supabase API by setting:

```bash
SUPABASE_URL=https://tafibcweszimipvhytft.supabase.co
SUPABASE_PUBLISHABLE_KEY=<your-publishable-key>
```

In this mode, the backend writes to the `invoices` table via Supabase client.

Before testing inserts, run SQL once in Supabase SQL Editor:

```sql
-- See full script in supabase/setup.sql
create table if not exists public.invoices (
	id bigint generated by default as identity primary key,
	invoice_number text not null,
	client_email text,
	sender_company_name text,
	total numeric(14, 2) default 0,
	currency text not null,
	template text not null,
	privacy_policy_accepted boolean not null default false,
	downloaded_at timestamptz,
	payload jsonb not null,
	created_at timestamptz not null default now()
);
```

For production, prefer `SUPABASE_SERVICE_ROLE_KEY` on backend instead of publishable key.

## Project Structure

- `/src/components` – React UI components
- `/src/utils` – helper functions for calculations and invoice data
- `/src/styles` – global CSS
- `/server` – Express API and PostgreSQL persistence

## Dependencies

- React
- Vite
- jsPDF (PDF export)
- SheetJS (`xlsx`) for Excel export
- react-signature-canvas for signature capture
- Express + pg for persistence

## Notes

This application uses localStorage to track invoice count, client emails and industries.  
After a user checks the privacy confirmation in preview and downloads the PDF, invoice data is also stored in PostgreSQL for future reference.  
The UI is intentionally minimal and modern without any third-party UI libraries.
